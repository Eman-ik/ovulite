  <name>Task 1: Backend auth — JWT, bcrypt, login endpoint, role middleware</name>
  <files>
    backend/app/auth/__init__.py
    backend/app/auth/security.py
    backend/app/auth/dependencies.py
    backend/app/api/__init__.py

    backend/app/schemas/__init__.py
    backend/app/schemas/auth.py
    backend/app/schemas/user.py
    backend/app/main.py
  </files>
  <action>
    Create `backend/app/schemas/auth.py`:
    - LoginRequest: Pydantic model with username (str) and password (str)
    - TokenResponse: access_token (str), token_type (str = "bearer")

    Create `backend/app/schemas/user.py`:
    - UserCreate: username, password, role, full_name, email (optional)
    - UserResponse: user_id, username, role, full_name, email, active, created_at (excludes password_hash)
    - UserInDB: extends UserResponse with password_hash (internal only)

    Create `backend/app/auth/security.py`:
    - `hash_password(password: str) -> str` — uses passlib bcrypt
    - `verify_password(plain: str, hashed: str) -> bool`
    - `create_access_token(data: dict, expires_delta: timedelta | None) -> str` — uses python-jose with HS256, reads SECRET_KEY and ALGORITHM from env
    - `verify_token(token: str) -> dict` — decodes and validates JWT, raises HTTPException 401 on failure
    - Token payload: {"sub": username, "role": role, "exp": expiry}

    Create `backend/app/auth/dependencies.py`:
    - `get_current_user(token: str = Depends(oauth2_scheme), db: Session = Depends(get_db)) -> User` — extracts user from JWT, queries DB to confirm user exists and is active
    - `require_role(*allowed_roles: str)` — returns a dependency that checks current_user.role is in allowed_roles, raises 403 if not
    - Use `OAuth2PasswordBearer(tokenUrl="/auth/login")` as the scheme

    Create `backend/app/api/auth.py`:
    - `POST /auth/login` — accepts LoginRequest (or OAuth2PasswordRequestForm), verifies credentials, returns TokenResponse
    - `POST /auth/register` — accepts UserCreate, creates user (admin-only in future, but open for initial setup)
    - `GET /auth/me` — returns current user info (requires auth)
    - `POST /auth/seed` — creates default admin user if no users exist (username: admin, password: ovulite2026). This is a one-time bootstrap endpoint.

    Update `backend/app/main.py`:
    - Include auth router: `app.include_router(auth_router, prefix="/auth", tags=["auth"])`
    - Add startup event to log "Ovulite API started"

    **Security notes:**
    - Use bcrypt with 12 rounds (passlib default)
    - JWT expiry: 30 minutes (from ACCESS_TOKEN_EXPIRE_MINUTES env var)
    - Never return password_hash in API responses
    - The /auth/seed endpoint should check if ANY user exists and refuse if so
  </action>
  <verify>curl -X POST http://localhost:8000/auth/seed; curl -X POST http://localhost:8000/auth/login -d "username=admin&password=ovulite2026"</verify>
  <done>POST /auth/login returns JWT for valid credentials; 401 for invalid; GET /auth/me returns user info with valid JWT; /auth/seed creates admin on empty DB</done>
</task>

    Create `frontend/src/components/ProtectedRoute.tsx`:

    - If authenticated, render children

    Update `frontend/src/pages/LoginPage.tsx`:

    - On successful login, navigate to /

    - Add loading state to button during login

    Update `frontend/src/App.tsx`:
